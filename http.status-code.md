# HTTP Status Codes for SOC Analysts

When analyzing web server logs in Splunk, you'll frequently encounter HTTP status codes.

Example query:

```spl
sourcetype=access_* | stats count by status
```

This counts how many times each HTTP status code appears.

---

## 200 - OK ✅

```text
200 = Request succeeded
```

### Example
A user visits:

```text
/index.html
```

The server successfully returns the page.

### SOC Notes
- Usually the most common status code.
- Indicates normal successful traffic.

---

## 400 - Bad Request ⚠️

```text
400 = Server couldn't understand the request
```

### Causes
- Malformed URL
- Invalid HTTP syntax
- Corrupted request

### SOC Notes
Large numbers of 400 errors may indicate:

- Vulnerability scanners
- Automated tools
- Malformed attack traffic

---

## 403 - Forbidden 🚫

```text
403 = Access denied
```

### Example

A user attempts to access:

```text
/admin
```

The resource exists, but permission is denied.

### SOC Notes

Attackers often generate many 403 errors while probing:

```text
/admin
/config
/backup
/private
```

---

## 404 - Not Found 📭

```text
404 = Resource does not exist
```

### Example

```text
/login.php
/admin.php
/backup.zip
```

The server cannot find the requested resource.

### SOC Notes

A few 404s are normal.

Large numbers may indicate:

- Web scanning
- Directory enumeration
- Vulnerability scanning

### Splunk Search

```spl
sourcetype=access_* status=404
| stats count by clientip uri_path
| sort - count
```

---

## 406 - Not Acceptable

```text
406 = Server cannot provide content matching request headers
```

### Example

The client requests a format the server cannot provide.

### SOC Notes

Less common.

Can sometimes appear during automated scanning.

---

## 408 - Request Timeout ⏳

```text
408 = Client took too long to send request
```

### Causes

- Slow network
- Client disconnected
- Network interruptions

### SOC Notes

Large spikes may indicate:

- Resource exhaustion
- Denial of Service (DoS) activity

---

## 500 - Internal Server Error 🔥

```text
500 = Server-side application error
```

### Causes

- Application crash
- Backend failure
- Misconfiguration

### SOC Notes

Sudden spikes can indicate:

- Application issues
- Exploitation attempts
- Malicious input causing crashes

### Splunk Search

```spl
sourcetype=access_* status=500
| stats count by clientip
| sort - count
```

---

## 503 - Service Unavailable 🚨

```text
503 = Service temporarily unavailable
```

### Causes

- Server overload
- Maintenance
- Backend service failure

### SOC Notes

May indicate:

- DoS attacks
- Infrastructure problems
- Capacity issues

---

## 505 - HTTP Version Not Supported

```text
505 = Unsupported HTTP version
```

### Causes

The client sends an unsupported HTTP version.

### SOC Notes

Often generated by:

- Scanners
- Malformed tools
- Incorrectly configured clients

---
